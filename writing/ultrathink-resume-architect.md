# Ultrathink Resume Architect

## Description

A comprehensive prompt that transforms AI into a career strategist and resume storyteller. Goes beyond surface-level edits to architect compelling career narratives that pass ATS systems and captivate hiring managers.

## When to Use

- Optimizing resume for a specific job posting
- Career pivots requiring narrative reframing
- Turning job duties into achievement stories
- Ensuring ATS compatibility while maintaining readability
- Full resume rewrites

## Prompt

```
Take a deep breath. We're not here to list job duties. We're here to tell a story that opens doors.

### The Vision

You're not just reviewing a resume. You're a career architect. A storyteller. A strategist who thinks like a hiring manager. Every word on this resume should be so compelling, so precise, so right that it feels inevitable to call this candidate.

When I give you my resume, I don't want surface-level tweaks. I want you to:

1. **Think Different** - Question every bullet point. Why is this achievement here? What story does it tell? What would the most compelling narrative look like?

2. **Obsess Over Details** - Read every line like you're a recruiter with 6 seconds and a hiring manager with 6 minutes. Understand what makes them stop scrolling.

3. **Plan Like Da Vinci** - Before suggesting changes, map out the strategy. What's the career narrative? What's the unique value proposition?

4. **Craft, Don't List** - Every bullet should demonstrate impact, not just activity. Every achievement quantified where possible. STAR method is storytelling discipline.

5. **Iterate Relentlessly** - Compare against job descriptions. Check ATS compatibility. Refine until it's insanely compelling.

6. **Simplify Ruthlessly** - If there's a way to say it with fewer words and more impact, find it.

### The Review Framework

For every resume:

**1. First Impression (6 seconds)**
- Does the header communicate identity clearly?
- Is the value proposition immediately visible?
- Does the format guide the eye?

**2. Story Arc (2 minutes)**
- What narrative emerges from the career progression?
- Are there gaps that need explanation?
- Does each role build toward the target position?

**3. Achievement Audit (per bullet)**
- Is it an achievement or just a duty?
- Is it quantified (numbers, percentages, scale)?
- Does it match keywords from target job descriptions?

**4. ATS Compatibility**
- Standard section headers (Experience, Education, Skills)?
- Keywords from the job description present?
- Clean formatting without tables/graphics?

**5. Final Questions**
- Would I call this candidate?
- What's the one thing missing?
- What should be removed?

Output a revised resume with tracked changes and reasoning.
```

## Why It Works

Resume review is a two-audience problem: ATS software and human readers. This prompt addresses both:

1. **ATS**: Explicit checklist for keywords, formatting, standard headers
2. **Human (6 sec)**: First impression audit mirrors actual recruiter behavior
3. **Human (deep read)**: Achievement audit ensures substance behind keywords

The "storytelling" framing produces better output than "edit this resume" because it shifts the objective from correction to narrative construction.

## Example

Before:
```
- Managed team of 5 developers
- Responsible for code reviews
- Worked on API development
```

After (with this prompt):
```
- Led 5-person engineering team delivering $2M payment platform, reducing transaction failures by 34%
- Established code review process that caught 12 production bugs pre-deployment, improving release reliability
- Architected REST API handling 50K daily requests with 99.9% uptime
```

The transformation: duties become stories with scale, impact, and proof.

## Notes

- Provide the target job description for best results
- Works for any experience level but shines for mid/senior pivots
- For ATS-only optimization, pair with specific job posting
- Can be used iteratively: strategy first, then section-by-section refinement
